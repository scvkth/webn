지옥에서 온 Git

  1. 커버 페이지

    2. 이 수업은 Git의 초심자에게는 기본적인 사용법을 중급자는 Git이 동작하는 원리를 소개해드리기 위한 수업입니다.
    이 수업에서는 명령어를 통해서 Git을 다루는 방법을 소개합니다. 명령어 기반 Git은 많은 시스템에 기본적으로
    설치 되어 있기 때문에 명령어로 Git을 다룰 수 있게 되면 많은 곳에서 특별한 설치 없이 Git을 사용할 수 있습니다.

  2. 버전관리의 본질

    3. 저는 기술에는 본질과 혁신이 있다고 생각합니다. 그 기술의 모든 제품이 공통적으로 가지고 있는 특성이 본질이고,
    특정 제품이 가지고 온 변화가 혁신이라고 생각합니다. 본 토픽의 하위에서는 버전관리 시스템을 관통하는 본질적인 기능을 살펴봅니다.
    본질은 최초의 혁신이면서 보편적인 기능이고 쉽습니다. 편한 마음으로 git을 즐기시길 바랄께요.

  4. 저장소 만들기

    5. 여기서는 프로젝트 환경을 만드는 법을 알아봅니다.

    1. 프로젝트 파일을 만듭시다.
      mkdir gitfth
    2. 프로젝트 디렉토리로 들어갑니다.
      cd gitfth
    3. 현재 디렉토리를 git의 (버전) 저장소로 만듭니다.
      git init

  5. git이 관리할 대상으로 파일 등록

    6. git은 기본적으로 새로운 파일을 관리하지 않습니다.
    파일을 관리하기 위해서는 관리 대상으로 등록을 해야합니다.

    1. 파일을 생성합니다.
      vim f1.txt
    2. git이 파일을 추적하도록 명령합니다.
      git add f1.txt
    3. 프로젝트 폴더의 상태를 확인합니다.
      git status

  6. 버전 만들기(commit)

    7. 버전에 포함될 버전을 만든 사람에 대한 정보를 설정합니다.
    이 설정은 ~/.gitconfig 파일에 저장되고 1번만 해주면 됩니다.

    1. git config --global user.name "자신의 닉네임"
    2. git config --global user.email "자신의 이메일"

  7. Stage area

    8. git은 commit 전에 add를 꼭해야 합니다. 그 이유는 선택적으로 파일을 버전에 포함시키기 위해서 입니다.
